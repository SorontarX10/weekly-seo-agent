# 1. Rola dokumentów w organizacji technologicznej

W dużej organizacji technologicznej dokumenty pełnią rolę:

-   nośnika decyzji
-   mechanizmu alignmentu
-   narzędzia redukcji niejasności
-   systemu odpowiedzialności

W kulturze technologicznej:

-   decyzje są data-driven
-   odpowiedzialność jest przypisana (ownership)
-   dokument zastępuje spotkanie

Zasada: \> Jeśli coś nie jest zapisane jasno -- nie istnieje.

------------------------------------------------------------------------

# 2. Model myślenia executive w tech

Executive w firmie technologicznej myśli w 4 wymiarach:

1.  Wpływ na użytkownika
2.  Wpływ finansowy
3.  Skalowalność technologiczna
4.  Ryzyko operacyjne

Każdy dokument powinien odpowiadać na te 4 pytania.

------------------------------------------------------------------------

# 3. Pyramid Principle -- pełny system

## 3.1 Zasada nadrzędna

Zaczynasz od rekomendacji.

## 3.2 Struktura

Główna Teza\
3 Argumenty\
Dowody (data / benchmark / eksperyment)

## 3.3 Przykład technologiczny

Rekomenduję migrację systemu X do architektury Y, ponieważ:

1.  Obniżamy koszt infrastruktury o 22%
2.  Skracamy time-to-market o 30%
3.  Redukujemy ryzyko downtime

------------------------------------------------------------------------

# 4. SCQA w środowisku data-driven

S -- Wzrost trafficu o 18%\
C -- Konwersja spada o 6%\
Q -- Gdzie tracimy użytkowników?\
A -- Checkout friction -- rekomenduję redesign

------------------------------------------------------------------------

# 5. MECE -- organizacja myślenia

W firmie technologicznej podział powinien być:

-   Acquisition
-   Activation
-   Retention
-   Monetization

Nie mieszamy warstw (np. kanałów z etapami funnelu).

------------------------------------------------------------------------

# 6. Decision Memo (Standard Tech Company)

Struktura:

1.  TL;DR (max 5 linii)
2.  Decyzja do podjęcia
3.  Rekomendacja
4.  3 Argumenty (MECE)
5.  Analiza danych
6.  Alternatywy
7.  Ryzyka
8.  Wpływ finansowy
9.  Wpływ technologiczny
10. Next steps

------------------------------------------------------------------------

# 7. Strategy Paper (Platform Company)

Strategia odpowiada:

-   Gdzie gramy?
-   Jak wygrywamy?
-   Czego nie robimy?
-   Jak skalujemy?

Filary (max 4): - Produkt - Technologia - Go-to-market - Monetizacja

------------------------------------------------------------------------

# 8. Annual Operating Plan (AOP)

Struktura:

1.  Ambicja roczna
2.  3--5 celów strategicznych
3.  Inicjatywy
4.  KPI
5.  Budżet
6.  Headcount
7.  Ryzyka
8.  Governance (QBR)

------------------------------------------------------------------------

# 9. Quarterly Business Review

1.  Wynik vs Plan
2.  Root cause analysis
3.  Korekty
4.  Prognoza
5.  Decyzje potrzebne od leadership

------------------------------------------------------------------------

# 10. Project Charter

Musi zawierać:

-   Problem statement
-   Success metrics
-   Scope / Out of scope
-   RACI
-   Milestones
-   Dependencies
-   Risk register

------------------------------------------------------------------------

# 11. Performance Updates

Nie raport aktywności. Raport wyniku.

Cel: +15% GMV\
Aktualnie: +11%\
Gap: 4%\
Plan domknięcia: X

------------------------------------------------------------------------

# 12. Dokumenty dla Zarządu

Zawierają:

-   Decyzję
-   Wpływ finansowy
-   Ryzyko
-   Trade-offs

Max 10 stron.

------------------------------------------------------------------------

# 13. Storytelling strategiczny

Narracja:

Rynek → Problem → Teza → Dowody → Plan → Wpływ

------------------------------------------------------------------------

# 14. Ownership Writing

Każdy dokument kończy się:

Owner:\
Deadline:\
Metric:

------------------------------------------------------------------------

# 15. Psychologia decyzji

Executive:

-   minimalizuje ryzyko
-   maksymalizuje wpływ
-   ceni klarowność

------------------------------------------------------------------------

# 16. Governance

Standard:

-   TL;DR obowiązkowe
-   3 argumenty max
-   KPI obowiązkowe
-   Owner obowiązkowy
-   Next steps obowiązkowe

------------------------------------------------------------------------

# 17. Executive Summary Standard

5 linii:

1.  Rekomendacja
2.  Dlaczego
3.  Wpływ
4.  Koszt
5.  Decyzja potrzebna

------------------------------------------------------------------------

# 18. Checklisty

✔ Czy jest teza?\
✔ Czy argumenty są MECE?\
✔ Czy są KPI?\
✔ Czy jest owner?\
✔ Czy są ryzyka?

------------------------------------------------------------------------

# 19. Antywzorce

❌ Brak decyzji\
❌ 20 KPI\
❌ Slajdy bez konkluzji\
❌ Brak właściciela

------------------------------------------------------------------------

# 20. AI w dokumentach managerskich

AI może:

-   pomóc w strukturze (Pyramid)
-   sprawdzić MECE
-   skrócić TL;DR
-   wygenerować checklistę ryzyk

AI nie zastępuje: - odpowiedzialności - decyzji - kontekstu biznesowego

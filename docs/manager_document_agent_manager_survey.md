# Ankieta dla Managerów: Ulepszenia Agenta Dokumentów

## Cel ankiety
Zebrać feedback od managerów, aby priorytetyzować rozwój Agenta Dokumentów pod realne potrzeby biznesowe.

## Metryczka respondenta
1. Jaki jest Twój obszar odpowiedzialności? (np. SEO, Produkt, Marketing, Operacje)
2. Jak często tworzysz dokumenty decyzyjne lub statusowe?
3. Jakie typy dokumentów tworzysz najczęściej? (np. management brief, roadmap, podsumowanie testu, notatka strategiczna)
4. W jakich językach najczęściej tworzysz dokumenty?

## Obecny proces i problemy
5. Jak dziś wygląda Twój proces tworzenia dokumentu od zera?
6. Co zajmuje najwięcej czasu w tym procesie?
7. Które etapy są najbardziej frustrujące? (zbieranie danych, struktura, redakcja, formatowanie, eksport, współpraca)
8. Jakie są najczęstsze błędy jakościowe w dokumentach?
9. Jak oceniasz spójność dokumentów między zespołami?

## Ocena obecnego Agenta Dokumentów
10. Jak oceniasz jakość generowanego outline’u? (1-10)
11. Jak oceniasz jakość pełnej redakcji AI? (1-10)
12. Jak oceniasz edycję fragmentów (selection rewrite)? (1-10)
13. Jak oceniasz użyteczność chatu planującego przed utworzeniem dokumentu? (1-10)
14. Jak oceniasz import dokumentów (DOCX/Google Doc)? (1-10)
15. Jak oceniasz eksport do DOCX/Google Docs? (1-10)
16. Która funkcja daje dziś największą wartość?
17. Która funkcja jest najmniej przydatna i dlaczego?

## Potrzeby funkcjonalne (priorytety)
18. Jakie 3 funkcje powinny być rozwijane w pierwszej kolejności?
19. Czy potrzebujesz szablonów dokumentów per typ dokumentu? Jeśli tak, jakich?
20. Czy potrzebujesz automatycznych checklist jakości przed finalizacją?
21. Czy potrzebujesz porównania wersji (diff) i komentarzy do zmian?
22. Czy potrzebujesz współedytowania dokumentu przez kilka osób?
23. Jak ważna jest automatyczna ekstrakcja kluczowych danych z załączników? (1-10)
24. Jak ważne są sugestie web research z akceptacją? (1-10)
25. Czy agent powinien proponować decyzje i rekomendacje „ready for approval”? (tak/nie + dlaczego)

## Integracje i środowisko pracy
26. Z jakich narzędzi musimy integrować agenta w pierwszej kolejności? (Google Docs, Drive, Slack, Jira, Confluence, inne)
27. Jak ważna jest pełna obsługa Google Docs jako „single source of truth”? (1-10)
28. Czy potrzebujesz automatycznego aktualizowania podpiętego Google Doca po każdej istotnej zmianie?
29. Jakie formaty wejścia/wyjścia są krytyczne poza obecnymi?

## UX i adopcja
30. Czy UI jest wystarczająco czytelne dla codziennej pracy? (1-10)
31. Które elementy UI są niejasne lub zbędne?
32. Jakiego sygnału postępu najbardziej brakuje podczas generowania/eksportu/syncu?
33. Jakie skróty lub automatyzacje przyspieszyłyby Twoją pracę?

## Zaufanie, jakość, governance
34. Na ile ufasz treści generowanej przez AI bez ręcznej korekty? (1-10)
35. Jakie mechanizmy zwiększą Twoje zaufanie? (źródła, cytowania, walidacja faktów, scoring jakości)
36. Jakie wymagania dot. poufności i bezpieczeństwa musimy spełnić?
37. Jakie logi/audyt zmian są potrzebne dla compliance?

## Pomiar sukcesu
38. Po czym poznamy, że agent realnie działa lepiej? (2-3 KPI)
39. O ile procent agent powinien skrócić czas przygotowania dokumentu, żeby był „must-have”?
40. Jaką jakość dokumentu uznajesz za akceptowalną dla managementu bez dużej ręcznej edycji?

## Pytania otwarte
41. Jaka jest jedna rzecz, którą agent powinien robić „genialnie”?
42. Czego agent absolutnie nie powinien robić?
43. Podaj przykład idealnego rezultatu końcowego dla Twojego typu dokumentu.
